# chp1 연습문제

1.Q1
  - (a) 디바이스의 센서와 액추에이터의 역할을 비교하여 설명하세요.

      센서(Sensor)와 액추에이터(Actuator)는 IoT 디바이스에서 상호 보완적인 역할을 수행한다.
      
      센서
      외부 환경의 물리적 정보를 감지하는 장치
      온도, 습도, 조도, 움직임 등의 데이터를 측정
      현실 세계 → 디지털 데이터로 변환하는 입력 장치
      액추에이터
      디지털 신호를 받아 실제 물리적 동작을 수행
      모터 구동, LED 점등, 스위치 제어 등 수행
      디지털 데이터 → 현실 세계의 변화로 출력하는 장치

  - (b) 디바이스와 대시보드에서 동일한 프로그래밍 언어를 사용하지 않는 이유를 설명하세요.

      디바이스와 대시보드는 운영 환경과 요구사항이 서로 다르기 때문이다.
      
      디바이스
      제한된 CPU, 메모리 사용 (임베디드 환경)
      저전력, 실시간 처리 중요
      주로 C, MicroPython 등 경량 언어 사용
      대시보드
      사용자 인터페이스(UI) 중심
      웹 기반 (브라우저에서 실행)
      JavaScript, HTML, Python 등 사용
    
  - (c) 브로커와 같은 정보 전달 전담 서버의 이점을 설명하세요.
    브로커(Broker)는 IoT 시스템에서 데이터를 중계하는 서버이다. 대표적으로 MQTT 브로커가 있다.

      이점:
      
      비동기 통신 가능
      송신자와 수신자가 동시에 연결되어 있지 않아도 데이터 전달 가능
      시스템 결합도 감소
      디바이스 간 직접 연결이 아닌 브로커를 통해 통신 → 구조 단순화
      확장성 향상
      디바이스가 증가해도 브로커만 관리하면 됨
      데이터 분배 기능
      하나의 데이터 → 여러 클라이언트에 전달 가능 (Pub/Sub 구조)
      신뢰성 향상
      QoS(Quality of Service)를 통해 메시지 전달 보장 가능

2. Q2
  - (a) Mosquitto와 Node-RED를 별도 서버에 설치할 경우의 이점을 설명하세요.
      Mosquitto(MQTT 브로커)와 Node-RED(데이터 처리 및 시각화)를 분리하면 다음과 같은 장점이 있다.
      
      - 성능 향상
          브로커와 처리 서버의 부하를 분산
      - 안정성 증가
          한 시스템 장애 시 다른 시스템 영향 최소화
      - 유지보수 용이
          각각 독립적으로 업데이트 및 관리 가능
      - 보안 강화
          네트워크 분리 및 접근 제어 가능
      - 확장성 확보
          트래픽 증가 시 특정 서버만 확장 가능
        
  - (b) Mosquitto와 Node-RED를 운영하기 위한 적절한 컴퓨팅 환경을 기술하세요.
        기본 요구 환경:
        
        OS: Linux (Ubuntu 권장), Windows, macOS 가능
        CPU: 최소 듀얼코어 이상
        RAM:
        소규모: 2GB 이상
        중규모 이상: 4GB 이상 권장
        저장공간: SSD 권장 (로그 및 데이터 처리 때문)
        
        구성 예시:
        
        Mosquitto:
        저사양 서버에서도 가능 (라즈베리파이 수준 가능)
        Node-RED:
        UI 및 흐름 처리 때문에 더 높은 자원 필요
