# Chapter 6 - IoT 및 하드웨어 기초 정리

---

## 1. IoT 디바이스 개발

### (a) 개발 보드 선정

- ESP32 선택
  - 저전력 (Deep Sleep 지원)
  - Wi-Fi / Bluetooth 내장
  - 배터리 기반 IoT에 적합

---

### (b) 배터리 절약 방법

- Deep Sleep → 최소 전력 유지
- Duty Cycle → 측정 → 전송 → Sleep 반복
- 이벤트 기반 → 변화 있을 때만 동작
- 통신 최소화 → 데이터 묶어서 전송

✔ 항상 동작 방식보다 전력 효율 훨씬 높음

---

### (c) 센서 연결 방식

#### ① 개별 핀
- 장점: 간단, 빠름
- 단점: 핀 많이 사용

#### ② OneWire
- 장점: 선 1개, 다중 연결
- 단점: 속도 느림

#### ③ I2C
- 장점: 핀 2개, 확장성 좋음
- 단점: 주소 충돌 가능

✔ 결론: ESP32 + Deep Sleep + I2C 구조 추천

## 2. ESP32 + 실드

### (a) 장점 / 한계

**장점**
- 빠른 개발
- 안정성 높음
- 회로 구성 불필요

**한계**
- 가격 높음
- 커스터마이징 제한

---

### (b) 실드 종류

- Wi-Fi / Bluetooth
- OLED / LCD
- 센서 실드
- 모터 드라이버
- 릴레이
- GPS
- 카메라

---

### (c) 컴팩트 모듈 10개

| 센서/모듈 | 용도 | 핀 |
|----------|------|----|
| DHT11 | 온습도 | GPIO4 |
| DS18B20 | 온도 | GPIO5 |
| BH1750 | 조도 | SDA(21), SCL(22) |
| MPU6050 | 가속도 | SDA(21), SCL(22) |
| OLED | 디스플레이 | SDA(21), SCL(22) |
| Relay | 전원제어 | GPIO23 |
| L298N | 모터 | GPIO18,19 |
| HC-SR04 | 거리 | 5,18 |
| MQ-2 | 가스 | GPIO34 |
| MicroSD | 저장 | 23,19,18,5 |

## 3. 브레드보드 분석

### (a) 브레드보드 vs PCB

| 구분 | 브레드보드 | PCB |
|------|------------|-----|
| 용도 | 테스트/실험 | 최종 제품 |
| 장점 | 재사용 가능, 납땜 불필요 | 안정성 높음 |
| 단점 | 접촉 불안정 | 수정 어려움 |

---

### (b) 크기별 분류

- Full Size (대형)
- Half Size (중형)
- Mini Size (소형)

---

### (c) M-F 점퍼 와이어 사용 예

- 센서 → ESP32 연결 시 사용
  - Female: 센서 핀
  - Male: 브레드보드

예:
- DHT11 → ESP32 연결
  - VCC → 3.3V
  - GND → GND
  - DATA → GPIO

✔ M-F는 모듈과 보드 연결에 가장 많이 사용됨

---

## ✔ 전체 정리

- ESP32는 저전력 IoT에 최적
- I2C 기반 확장이 가장 효율적
- 브레드보드는 테스트용, PCB는 완성용
